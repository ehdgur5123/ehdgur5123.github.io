# 트래픽(traffic)

**트래픽**<sup>traffic</sup>이란, 네트워크 내의 정보량의 의미

특정 노드에 트래픽이 몰린다는 것은 해당 노드가 특정 시간 동안 처리해야 할 정보가 많음을 의미

트래픽이 몰려 특정 노드에 **과부하**<sup>overhead</sup>가 생기면, 네트워크의 성능 저하가 발생

# 네트워크 성능 지표

네트워크의 성능을 평가할 수 있는 세 가지 대중적인 지표로는 **처리율**, **대역폭**, **패킷 손실**이 있습니다.

## 처리율(throughput)

**단위 시간당 네트워크를 통해 실제로 전송되는 정보량**을 의미

처리율의 단위는 <b>bps</b>(bit/s), <b>Mbps</b>(Mbit/s), <b>Gbps</b>(Gbit/s) 또는 초당 패킷 수를 표현하는 <b>pps</b>(p/s)의 단위를 사용

처리율은 비교적 실시간성이 강조된 지표로, 특정 노드가 얼마만큼의 트래픽을 처리하는 중인지 판단하기 위해 주로 사용됨

## 대역폭(bandwidth)

네트워크 성능 측정 영역에서의 **대역폭**이란, **단위 시간 동안 통신 매체를 통해 송수신할 수 있는 최대 정보량**을 의미

> 신호 처리 영역에서의 <b>대역폭</b>은 주파수의 범위를 의미

단위는 처리율과 같은 <b>bps</b>, <b>Mbps</b>, <b>Gbps</b>의 단위를 사용

통신 매체가 <u>높은 대역폭을 가지고 있다는 것</u>은 <u>많은 정보를 송수신할 능력이 있음</u>을 의미

## 패킷 손실(packet loss)

**패킷 손실**은 송수신되는 패킷이 손실된 상황을 의미

전체 패킷 중 유실된 패킷을 백분위로 표현한 값을 사용하는 경우가 많음

리눅스의 `ping` 명령어를 통해 '패킷 손실율'을 알 수 있음(`0.0% packet loss`)
